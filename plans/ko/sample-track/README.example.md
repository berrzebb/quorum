# 설계: Sample Track

> 상태: `planned` | 범위: `improved/sample-track`

## 목적

이 트랙이 해결하는 문제와 목표를 1~3문장으로 기술한다.
"무엇을 추가"가 아니라 "어떤 경계/구조를 고정"하는지에 집중한다.

## 현재 기준점

- 현재 코드베이스에서 이 트랙과 관련된 상태를 기술
- 이미 완료된 부분과 남아 있는 문제를 분리

## 수행 범위

- 이 트랙에서 다루는 항목 목록
- 각 항목은 코드 경계 또는 책임 단위로 표현

## 제외 범위

- 이 트랙에서 명시적으로 다루지 않는 것
- 인접 트랙으로 위임하는 항목

## 완료 기준

- 구체적인 동작 기준 (예: "X 경로에서 Y 검증이 통과함")
- 테스트 또는 lint 결과로 확인 가능한 기준
- 추상적 기준("개선됨") 금지
