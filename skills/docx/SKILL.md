# DOCX Document Generator

Create professional Word documents from code analysis results, markdown reports, or structured data.

## Quick Reference

| Task | Approach | Detail |
|------|----------|--------|
| Markdown → DOCX | `python scripts/create_docx.py --md report.md -o out.docx` | Full conversion with styling |
| JSON → DOCX | `python scripts/create_docx.py --json data.json -o out.docx` | Structured input |
| With template | Add `--template brand.docx` to either | Custom branding/styles |
| Read existing | `python-docx` `Document()` | Read `references/reference.md` |

## Workflow

1. **Gather content** — run analysis tools, collect data, or receive user input
2. **Choose input format**:
   - **Markdown** (preferred): write content as `.md`, convert via script
   - **JSON**: build structured data when you need precise control over sections
3. **Run the script** to generate DOCX
4. **Verify** — check the output exists and report the file path

## Dependencies

```bash
pip install python-docx   # Core (usually pre-installed)
```

Optional for diagram rendering:
```bash
npm install -g @mermaid-js/mermaid-cli   # mmdc renders mermaid → PNG
```

If `mmdc` is not available, Mermaid diagrams are included as formatted code blocks with a note.

## Markdown Input (Recommended)

Write a standard markdown file. The script handles:

- `# H1` through `#### H4` → Word heading styles
- Paragraphs with **bold**, *italic*, `inline code`
- Fenced code blocks with language tag → monospace with background
- ```` ```mermaid ```` blocks → rendered diagram (mmdc) or code block fallback
- `| table |` syntax → styled Word tables
- `- bullet` and `1. numbered` lists
- `---` → horizontal rule
- `> blockquote` → indented italic

The first `# H1` becomes the title page. Add metadata via CLI flags:

```bash
python skills/docx/scripts/create_docx.py \
  --md report.md \
  --title "Architecture Analysis" \
  --author "Generated by Claude" \
  --date "2026-03-27" \
  -o report.docx
```

## JSON Input (For Precise Control)

When you need exact control over document structure:

```json
{
  "title": "Code Analysis Report",
  "subtitle": "quorum v0.4.6",
  "author": "Generated by Claude",
  "date": "2026-03-27",
  "toc": true,
  "sections": [
    {
      "heading": "Executive Summary",
      "level": 1,
      "content": "This report covers the architecture and quality metrics..."
    },
    {
      "heading": "Architecture",
      "level": 1,
      "content": "The system follows a layered architecture.",
      "mermaid": "flowchart LR\n  A[CLI] --> B[Bus]\n  B --> C[Providers]",
      "diagram_caption": "System Architecture"
    },
    {
      "heading": "Quality Metrics",
      "level": 2,
      "table": {
        "headers": ["Metric", "Score", "Status"],
        "rows": [
          ["Type Safety", "0.85", "PASS"],
          ["Test Coverage", "0.72", "WARN"]

    },
    {
      "heading": "Key Implementation",
      "level": 2,

    {
      "heading": "Action Items",
      "level": 1,
      "bullets": ["Increase test coverage to 80%", "Fix type errors in providers/"]
    }
  ]
}
```

## Document Styling

The script applies a professional default style:

| Element | Font | Size | Color |
|---------|------|------|-------|
| Title | Calibri | 36pt | #1E2761 (dark blue) |
| Subtitle | Calibri | 18pt | #065A82 (ocean blue) |
| Heading 1 | Calibri Bold | 24pt | #1E2761 |
| Heading 2 | Calibri Bold | 18pt | #1E2761 |
| Body | Calibri | 11pt | Black |
| Code | Consolas | 9pt | Black on #F5F5F5 |
| Table header | Calibri Bold | 10pt | White on #1E2761 |

## Template System

Use `--template` to apply a custom `.docx` as the base document. The template's styles (fonts, colors, margins, headers/footers) are preserved:

```bash
python scripts/create_docx.py --md report.md --template corporate.docx -o report.docx
```

Template tips:
- Create a `.docx` with your desired styles for Heading 1-4, Normal, List Bullet, etc.
- The script inherits all styles from the template
- Headers/footers in the template are preserved
- Read `references/templates.md` for detailed template creation guide

## Mermaid Diagrams

Mermaid diagrams are supported in both input formats:

- **Markdown**: use ```` ```mermaid ```` fenced blocks
- **JSON**: use `"mermaid": "flowchart LR\n  ..."` in a section

Rendering priority:
1. `mmdc` (mermaid-cli) → PNG image embedded in DOCX
2. Fallback → formatted code block with "(install mmdc for image rendering)" note

For diagram syntax, reference `skills/mermaid/references/` — particularly `flowchart.md`, `sequence.md`, `architecture.md`.
