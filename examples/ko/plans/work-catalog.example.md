# 설계: Improvement Work Catalog

> **상태**: `planned` | **유형**: 작업 카탈로그

## 분류 기준

각 작업 단위는 아래 기준으로 분류한다.

- **Track**: foundation / execution / platform / support
- **Type**: local refactor / cross-cutting / migration / policy / testing
- **Model**: `Sonnet 4.6` / `Opus 4.6` / `Opus -> Sonnet`
- **Risk**: low / medium / high
