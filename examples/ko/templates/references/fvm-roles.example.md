# FVM 역할 및 해석 가이드

## 목적

이 레퍼런스는 기능 검증 매트릭스(FVM)의 역할 계층과 해석 규칙을 정의한다. 스카우트는 예상 접근 패턴을 이해하는 데, 밸리데이터는 실패를 분류하는 데 사용한다.

## 역할 계층

가시성 등급은 낮은 제한에서 높은 제한 순서:

```
public < authenticated < team_member < team_manager < team_owner < superadmin
```

### 등급 × 역할 매트릭스

| 등급 | 미인증 | viewer | member | manager | owner | superadmin |
|------|:-:|:-:|:-:|:-:|:-:|:-:|
| public | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| authenticated | ❌ | ✅ | ✅ | ✅ | ✅ | ✅ |
| team_member | ❌ | ✅ | ✅ | ✅ | ✅ | ✅ |
| team_manager | ❌ | ❌ | ❌ | ✅ | ✅ | ✅ |
| team_owner | ❌ | ❌ | ❌ | ❌ | ✅ | ✅ |
| superadmin | ❌ | ❌ | ❌ | ❌ | ❌ | ✅ |

- ✅ = 예상 HTTP 200 (또는 유효한 리소스 접근 시 2xx/404)
- ❌ = 예상 HTTP 401 (미인증) 또는 403 (인증됨 + 권한 부족)

### 역할 정의

| 역할 | 시스템 역할 | 팀 역할 | 범위 |
|------|-----------|---------|------|
| superadmin | superadmin | — | 전체 시스템, 모든 등급 통과 |
| owner | user | owner | 팀 소유자, 설정·프로바이더 관리 |
| manager | user | manager | 팀 관리자, 채널·시크릿·멤버 관리 |
| member | user | member | 팀 멤버, 워크플로우 생성·워크스페이스 사용 |
| viewer | user | viewer | 읽기 전용, 조회만 가능 |
| 미인증 | — | — | 세션 없음, 공개 페이지만 접근 |

### 특수 케이스

- **인증 비활성**: 서버에서 인증이 비활성화된 경우 `public` 외 모든 등급이 `public`으로 취급 (싱글유저 모드). FVM 검증 시 인증 기반 행은 건너뜀.
- **superadmin 우회**: superadmin은 팀 소속 여부와 무관하게 `team_*` 등급 포함 모든 등급을 통과.
- **view vs manage**: 각 라우트에 두 개의 등급이 있음. `view`는 페이지 접근(GET/읽기), `manage`는 변경 작업(POST/PUT/PATCH/DELETE). FVM은 GET에 `view` 등급, 변경에 `manage` 등급 적용.

## FVM 실패 분류

| 코드 | 의미 | 심각도 | 조치 |
|------|------|--------|------|
| AUTH_LEAK | 정책상 거부해야 하지만 서버가 2xx 반환 | 🔴 긴급 | 보안 수정 — 인가 가드 추가 필요 |
| FALSE_DENY | 정책상 허용해야 하지만 서버가 403/401 반환 | 🟡 높음 | 버그 수정 — 미들웨어/핸들러 인증 로직 확인 |
| ENDPOINT_MISSING | 정의된 엔드포인트에 404 반환 | 🟠 중간 | 라우트 등록 또는 엔드포인트 경로 확인 |
| PARAM_ERROR | 유효한 엔드포인트에 400/422 반환 | 🟢 낮음 | 요청 본문 스키마 확인, 테스트 데이터 문제일 수 있음 |

### 동적 엔드포인트 해석

경로 파라미터가 있는 엔드포인트(예: `/api/admin/teams/:id`)는 플레이스홀더 ID로 호출. 예상 동작:
- **200**: 엔드포인트 정상 (리소스 존재 여부 무관)
- **404**: 엔드포인트 정상, 리소스 미존재 (`:id` 라우트에서 허용)
- **403/401**: 접근 거부 (미인가 역할에 대해 예상)
- **400**: 파라미터 형식 오류 (플레이스홀더 값에 대해 허용 가능)

## RTM과의 통합

FVM은 RTM을 보완 — 서로 다른 검증 레이어:

| 측면 | RTM | FVM |
|------|-----|-----|
| 수준 | 소스 코드 | HTTP 런타임 |
| 검증 대상 | 코드 존재, 테스트 통과 | API 계약 동작, 인가 적용 |
| 생성 주체 | 스카우트 (code_map, grep) | fvm_generate (정적 분석) |
| 검증 주체 | 구현자 (vitest) | fvm_validate (HTTP 요청) |
| 실패 유형 | 구현 갭 | 통합/보안 갭 |

프론트엔드 라우트가 범위에 포함될 때 스카우트는 RTM과 FVM을 모두 생성해야 한다.
