# i18n Checker — Domain Knowledge

**Primary tool**: `i18n_validate`

## Focus Areas
1. **Locale key parity** — all keys present in every locale file
2. **Hardcoded strings** — user-visible text not using i18n system
3. **Interpolation format** — consistent placeholder syntax
4. **Pluralization** — proper plural forms for all locales
5. **RTL support** — layout considerations for RTL languages

## Checklist
- [ ] I18N-1: New UI strings use locale keys (no hardcoded text)
- [ ] I18N-2: Keys exist in ALL locale files (ko.json AND en.json)
- [ ] I18N-3: Interpolation placeholders match across locales
- [ ] I18N-4: Date/number formatting uses locale-aware APIs
- [ ] I18N-5: No string concatenation for translated text

## Rejection Codes
- **i18n-parity**: Locale key missing in one or more locale files
- **i18n-hardcoded**: User-visible string not internationalized
