# Accessibility Auditor — Domain Knowledge

**Primary tool**: `a11y_scan`

## Focus Areas
1. **WCAG 2.1 AA compliance** — contrast ratios, text alternatives, focus management
2. **ARIA attributes** — correct roles, states, properties
3. **Semantic HTML** — proper heading hierarchy, landmark regions
4. **Keyboard support** — tab order, focus indicators, skip links
5. **Screen reader** — live regions, form labels, error announcements

## Checklist
- [ ] A11Y-1: All interactive elements keyboard-accessible
- [ ] A11Y-2: ARIA roles and attributes correctly applied
- [ ] A11Y-3: Color contrast meets AA minimum (4.5:1 text, 3:1 large)
- [ ] A11Y-4: Form inputs have associated labels
- [ ] A11Y-5: Focus management after dynamic content changes
- [ ] A11Y-6: Images have meaningful alt text (or empty alt for decorative)

## Rejection Codes
- **a11y-regression**: Existing accessibility degraded
- **a11y-gap**: New component missing accessibility support
